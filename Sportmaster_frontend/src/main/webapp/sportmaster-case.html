📌 Почему это критично для AI-summary: 82,3% моделей имеют рейтинг ⩾4★, и только 6,3% — ниже 3★. Но психологические исследования показывают, что негативная информация «весит» для восприятия сильнее положительной — эффект negativity bias, «bad is stronger than good» (Baumeister et al.). Покупатели целенаправленно ищут единичные отрицательные отзывы перед покупкой, даже когда позитив в явном большинстве. AI-summary не имеет права «размывать» редкий, но content-rich негатив общим позитивным фоном — иначе саммари будет восприниматься как нечестное. Источники: Rethinking the Negativity Bias (NIH / PMC), Colorado State University — Why negative reviews impact customers more.

◆ Вход: сырые данные

  • Текст отзыва, оценка, дата, ID товара
  • Верифицированная покупка (флаг)
  • Реакции: полезно / бесполезно
  • Фото-отзывы (метаданные)

◆ Обработка: логика

  • Фильтр: min 3 отзыва на товар
  • Порог доверия по спам-скору
  • Взвешивание свежих отзывов
  • Инкрементальное обновление раз в сутки

◆ Выход: продукт

  • Саммари 2–3 предложения
  • 3–5 плюсов и минусов
  • Аспектные теги с тональностью
  • Ссылки на репрезентативные отзывы
# Упрощённая схема пайплайна
def build_ai_summary(product_id):
    reviews = load_reviews(product_id)              # сбор из БД
    reviews = clean(reviews)                        # дедуп + антиспам + нормализация
    if len(reviews) < 3: return None            # порог достаточности

    aspects = absa_model(reviews)                 # тональность по аспектам
    summary = llm_summarize(                       # генерация LLM
        reviews, aspects,
        prompt="Собери честное саммари: суть, плюсы, минусы"
    )
    return cache(product_id, summary)           # кэш → карточка товара
